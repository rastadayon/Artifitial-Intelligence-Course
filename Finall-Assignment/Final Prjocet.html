Predicting Cellphone Device Prices Based on Real Dataset¶

Breif Description of the Project¶

In this final project the objective is predicting the price of cellphone devices using a real dataset provided by Divar. The dataset can be seen here. Divar is an online shop which people can put their goods for sale. This dataset is in persian and has the following columns:

  • brand : This feature specifies the device brand. this field consists of categorical values.

  • city : The city where the ad was published. this field consists of categorical values.

  • title : Shows the title of the ad. Each ad has a title and a description field. This field is in Persian.

  • desc : Shows the description of the ad. This field is in Persian as well.

  • image_count : The number of images uploaded for the ad by the ad owner. This field consists of numeric values.

  • created_at : The day and hour in which the ad was created. This field is in English and has string values.

  • price : The price of the device set by the ad owner. This field consists of numeric values but they are not all consistant because they have been set by the users. Some of the values are specified by -1. Thes values will be predicted in the end with the best model that has been achieved.

def fix_price(dataset):
    dataset['price'] = dataset['price'].apply(lambda x : 1000*x if (25 <= x <= 3000) else x)
    dataset['price'] = dataset['price'].apply(lambda x : None if ((0 <= x < 25) or (3000 < x < 25000)) else x)
    dataset = dataset.replace(to_replace='None', value=np.nan).dropna()
    return dataset

In this section of the project Lemmatize was applied to the data since we saw in the previous projects the advantages it had over Stemming.

For applying lemmitize to Persian text Lemmatizer() from the Hazm library was used.

Dataset division¶

After the normalization is done the datafram is divided into 3 section. The first one is the evaluation set which are the devices with price equal to -1. The second and thirs sections are training and test sets. All the trainings and information gatherings will only happen on the training set. The division of the dataset is being done in this part because in the next we want to gather which words are important are informative so they can be used as features and we do not want to have access to the test set data for that.

For training and test sets 0.8 of each brand are put into the trainig set and the rest are put into the test set. The reason for this is that we want to have as much of a uniform division as possible since having a training set that has vrey little information on one brand and a lot of information on another brand may cause the model to have a more accurate on one brand than on another one.

def divide(dataset, percentage = 0.8):
   
    dfs = []
    trainDf = pd.DataFrame()
    testDf = pd.DataFrame()
    data = dataset[dataset['price'] != -1]
    data = data.reset_index(drop=True)
    brands = list(data['brand'].unique())
    for brand in brands:
        df =  data[data['brand'] == brand].copy()
        msk = np.random.rand(df.shape[0]) < percentage
        dfList = []
        dfList.append(df[msk])
        dfList.append(df[~msk])
        dfs.append(dfList)
    for df in dfs:
        trainDf = pd.concat([trainDf,df[0]]).sort_index()
        testDf = pd.concat([testDf, df[1]]).sort_index()
    
    return trainDf, testDf

Creating new features¶

The features given in the dataset are very few (city, created_at, brand) and it is quite obvious that with those features only a good model cannot be trained and features should be extracted from the title and description columns.

For creating features we have to decide which words are most informative and use them as features.

def get_words_counts(data, column):
    train_data = data[data['price'] != -1]
    desc = train_data[column]
    all_descs = []
    for item in desc:
        all_descs += item
    word_counts = {}
    for item in all_descs:
        if(item in word_counts):
            word_counts[item] += 1
        else:
            word_counts[item] = 1
    return word_counts

To find the best features to add to the dataset the TF-IDF algorithm is used. The dictionaries that were set in the above code section represent the TF part of the algorithm which represents the Term Frequency which means how many times each term has repeated in the entire documents. This term should be multiplied by the IDF which stands for Inverse Document Frequency. IDF represents the uniqueness of the words which relate to how informative they are. For example the word 'hello' () is repeated quite a lot but it is not necessarily informative about the device in question.

The IDF term is calculated using the following equation:

$$ IDF = ln(\frac{Total-Number-of-Documents}{Number-of-Documents-with-t-in-it})$$

The following function calculates the TD-IDF values.

In [820]:
def calc_number_of_docs(t, docs):
    num = 0
    for item in docs:
        if(t in item):
            num += 1
    return num
def calc_tfidf(desc_tf, title_tf, dataset):
    descs = list(dataset['desc'])
    titles = list(dataset['title'])
    N_descs = len(descs)
    N_title = len(titles)
    desc_tfidf = {}
    title_tfidf = {}
    for item in desc_tf:
        item_idf = math.log(N_descs/calc_number_of_docs(item, descs)) 
        desc_tfidf[item] = desc_tf[item]*item_idf
    for item in title_tf:
        item_idf = math.log(N_title/calc_number_of_docs(item, titles))
        title_tfidf[item] = title_tf[item]*item_idf
    
    return desc_tfidf, title_tfidf

After calculating the tf-idf values for all the tokenized words in the description and title columns the words are sorted based on their tf-idf values and 200 words with the highest tf-idf values are selected from the description column to be new features and 100 words from the title column are selected in the same way.

def add_new_features (desc_features, title_features, dataset):
    data = dataset.copy()

Since a model can not understand categorical values these values should be encoded. Two ways of encoding these values are: Label encoding and One-Hot encoding. Each of these methode have advantages and disadvantages but since the label encoding methode introduces an ordering into the data and two out of 4 models trained for this project (linear regression and K Nearest Neighbour) are sensetive to the ordering implies by the label encoding, I decided to use one-hot encoding. For one-hot encoding the get_dummies() function of the pandas library was used.

An important thing to note is the Performance Metric. Since this is a regression project, accuracy and metrics like it can not describe the model. Instead metrics that describe the error can come in handy. Three metrics can be calculated, the first and second one were used in the project.

  1. Root Mean Squared Error (RMSE) $$ RSME = \sqrt{ 1/n \sum_{j=1}^{n}(y_j - \hat{y_j})^2}$$

  2. Mean Absolute Error (MAE)

$$ MAE = 1/n \sum_{j=1}^{n}|y_j - \hat{y_j}|$$
  1. Root Mean Squared Logarithmic Error (RMSLE)
$$ RMSLE = \sqrt{ 1/n \sum_{j=1}^{n}(log(y_j) - log(\hat{y_j}))^2}$$

Each of these metrics are commenly used metrics and have their own advantage and disadvantages. The first two metrics express average model prediction error in units of the variable of interest. Both metrics can range from 0 to and are indifferent to the direction of errors. However since in RMSE errors are squared before they are averaged, it gives a relatively high weight to large errors. This means the RMSE should be more useful when large errors are particularly undesirable. In the later sections it is observable that the RMSE is higher in value.

RMSLE has the meaning of a relative error, while RMSE is an absolute error. RMSE is more prone to getting larger faster when it encounters an outlier. In contrast, even on the introduction of the outlier, the RMSLE error is not affected much. Also RMSLE considers a higher penalty for over estimation rather than under estimation.

KNN Regressior¶

The KNN algorithm uses feature similarity to predict the values of any new data points. This means that the new point is assigned a value based on how closely it resembles the points in the training set. The algorithm takes the K nearest neighbours to the new datapoint and sets the average of their prices as the prediction.

Since KNN relies on closeness using Label encoding could have resulted in wrong predictions as it makes the categorical values into numerical values that have some sort of an ordering and when calculating the distance between the points the ordering could create an unreal distance between certain points.

LightGBM Regressor¶

According to this link Light GBM is a gradient boosting framework that uses tree based learning algorithm. Light GBM grows tree vertically while other algorithm grows trees horizontally meaning that Light GBM grows tree leaf-wise while other algorithms such as decision tree grows level-wise. It will choose the leaf with max delta loss to grow. When growing the same leaf, Leaf-wise algorithm can reduce more loss than a level-wise algorithm.
